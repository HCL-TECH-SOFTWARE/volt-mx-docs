/**@class android.app.backup.BackupAgentHelper

 A convenient {@link android.app.backup.BackupAgent} wrapper class that automatically manages
 heterogeneous data sets within the backup data, each identified by a unique
 key prefix.  When processing a backup or restore operation, the BackupAgentHelper
 dispatches to one or more installed {@link android.app.backup.BackupHelper} objects, each
 of which is responsible for a defined subset of the data being processed.
 <p>
 An application will typically extend this class in its own
 backup agent. Then, within the agent's {@link android.app.backup.BackupAgent#onCreate() onCreate()}
 method, it will call {@link #addHelper(String, android.app.backup.BackupHelper) addHelper()} one or more times to
 install the handlers for each kind of data it wishes to manage within its backups.
 <p>
 The Android framework currently provides two predefined {@link android.app.backup.BackupHelper} classes:</p>
 <ul><li>{@link android.app.backup.FileBackupHelper} - Manages the backup and restore of entire files
 within an application's data directory hierarchy.</li>
 <li>{@link android.app.backup.SharedPreferencesBackupHelper} - Manages the backup and restore of an
 application's {@link android.content.SharedPreferences} data.</li></ul>
 <p>
 An application can also implement its own helper classes to work within the
 {@link android.app.backup.BackupAgentHelper} framework.  See the {@link android.app.backup.BackupHelper} interface
 documentation for details.
